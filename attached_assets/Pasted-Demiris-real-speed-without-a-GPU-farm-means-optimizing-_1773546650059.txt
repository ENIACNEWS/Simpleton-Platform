## 3. Quantization Without Quality Loss

| Precision | Memory Reduction | Speed Gain |
|-----------|------------------|------------|
| FP16 | 1x | 1x |
| INT8 | 2x | 2-3x |
| INT4 (AWQ/GPTQ) | 4x | 3-4x |
| INT4 + Marlin kernel | 4x | **4-6x** |

**Current best practice:**
- Use AWQ or GPTQ quantization (not naive INT4)
- Marlin kernels for inference (2-4x speedup over standard INT4)
- Keep key layers at higher precision (first/last layers, attention)

**For your use case:** Quantize the specialized models (diamond calculator, metal pricing) aggressively. Keep the general chat model at higher precision.

## 4. CPU Offloading & Hybrid Execution

Don't let GPU memory be the bottleneck.

| Strategy | When to Use |
|----------|-------------|
| Layer-wise offloading | Models larger than VRAM |
| Prompt processing on CPU | Long contexts, cheap compute |
| Speculative CPU draft | CPU runs tiny draft model |

**Implementation:** vLLM or TensorRT-LLM handle this automatically. Use them.
