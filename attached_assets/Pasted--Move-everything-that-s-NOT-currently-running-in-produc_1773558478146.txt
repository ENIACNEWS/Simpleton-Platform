### Code Style Rules:

- Remove any obvious AI comments like "// This function does X"
- Replace with meaningful comments that explain WHY, not WHAT
- Add JSDoc comments for all public functions
- Remove debug logs, console.logs, test code
- Ensure error handling is present (no silent failures)

### Structure Rules:
- Group related files in feature folders (not all in one place)
- Keep API routes organized by resource
- Keep database logic in `/db` folder
- Keep business logic in `/services` folder
- Keep types/interfaces in `/types` folder

### Make It Look Human:
- Vary comment length and style slightly (humans aren't perfectly consistent)
- Add occasional "TODO" comments for future improvements
- Include a few minor imperfections (a real codebase always has them)
