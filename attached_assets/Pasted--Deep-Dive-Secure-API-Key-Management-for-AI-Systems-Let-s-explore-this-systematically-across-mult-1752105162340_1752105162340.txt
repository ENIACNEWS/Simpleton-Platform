# Deep Dive: Secure API Key Management for AI Systems

Let's explore this systematically across multiple dimensions:

## 1. Threat Modeling for API Keys

**Attack Vectors to Protect Against:**
- Source code repository leaks (GitHub, GitLab, etc.)
- Server environment inspection (env vars, process listings)
- Network interception (MITM attacks)
- Internal misuse (overprivileged employees)
- Supply chain attacks (compromised dependencies)

## 2. Architectural Patterns

### Layered Defense Approach:
```mermaid
graph TD
    A[Raw Key] --> B[Key Encryption]
    B --> C[Secure Storage]
    C --> D[Controlled Access]
    D --> E[Usage Monitoring]

```

### Runtime Injection Models:
- **Sidecar Pattern**: Keys delivered via secure sidecar container
- **Service Mesh**: Istio/Linkerd with secret distribution
- **Ephemeral Tokens**: Short-lived JWTs derived from master key

## 4. Cryptographic Protections

**Envelope Encryption Pattern:**
1. Generate DEK (Data Encryption Key) per request
2. Encrypt payload with DEK
3. Encrypt DEK with KEK (Key Encryption Key) from HSM
4. Store encrypted DEK with encrypted payload

## 5. Runtime Protection Mechanisms

**Memory Handling Best Practices:**
```c
// Example in C for sensitive memory handling
void use_key(char* key) {
    volatile char* secure_key = malloc(KEY_LENGTH);
    memcpy(secure_key, key, KEY_LENGTH);
    // Use key
    memset(secure_key, 0, KEY_LENGTH); // Immediate cleanup
    free(secure_key);
}
```

Python equivalent with ctypes:
```python
from ctypes import create_string_buffer, memset
import sys

def secure_key_usage(key):
    secure_buf = create_string_buffer(key.encode())
    try:
        # Use key
        pass
    finally:
        memset(secure_buf, 0, len(key))
```

## 7. Compliance Considerations

**Key Rotation Strategies:**
- Time-based (e.g., every 90 days)
- Usage-based (after X calls)
- Compromise-based (immediate rotation)
- Versioned keys (allow overlap during transition)

## 8. Monitoring and Anomaly Detection

Example Detection Rules:
- Geographic anomalies (unexpected countries)
- Usage spikes (2x normal rate)
- Unusual times (outside business hours)
- Multiple failed attempts
